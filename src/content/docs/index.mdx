## Why Tulpar

<CardGrid stagger>
	<Card title="C-class performance" icon="rocket">
		AOT-compiled via LLVM. Lands in the **1.3–1.4× C** range on tight loops on identical hardware — same neighbourhood as Rust and Go. HTTP server hits ~24k req/sec single-threaded, ~Node.js parity.
	</Card>
	<Card title="API on day one" icon="seti:json">
		`import "wings"`, register a route, `serve(8080)` — done. Built-in `/healthz`, `/metrics` (JSON + Prometheus), OpenAPI 3.0 + Swagger UI auto-gen, schema validation, dependency injection, structured logging, keep-alive, multi-threaded request handling.
	</Card>
	<Card title="Modern editor tooling" icon="laptop">
		Bundled language server: diagnostics, hover, completion, go-to-definition, find-references, rename. Plus `tulpar fmt` (gofmt-style idempotent formatter) and a VS Code extension that just works.
	</Card>
	<Card title="Package manager" icon="puzzle">
		`tulpar pkg init / add / install / list / remove`, `tulpar.toml` manifest, `tulpar.lock` lockfile. Vendor local paths, fetch single-file URLs, or pull from a registry by semver.
	</Card>
	<Card title="Batteries included" icon="seti:db">
		Embedded SQLite + Active-Record ORM. Outbound HTTP/HTTPS client. Regex, CSV, file glob, datetime (ISO 8601 parse/format/weekday), JSON, threads, mutex, sockets — all in the runtime, no `npm install`.
	</Card>
	<Card title="UTF-8 · Türkçe · English" icon="pencil">
		Identifiers, strings, paths — all UTF-8. Compiler error messages are bilingual (Turkish + English via `tr_en` localisation). Cross-platform: Linux, macOS, Windows.
	</Card>
</CardGrid>

## From zero to a REST API

A real, persistent CRUD API — model, routes, server — in one file. The server
boots with `/healthz`, `/metrics`, `/openapi.json` and a Swagger UI for free.

```tulpar
import "wings";
import "orm";

orm_open("app.db");
define_model("users", {
    "id":   "INTEGER PRIMARY KEY AUTOINCREMENT",
    "name": "TEXT NOT NULL",
    "age":  "INTEGER"
});

func list_users(req)  { return ok(orm_all("users")); }

func create_user(req) {
    int id = orm_create("users", req.json);
    return created(orm_find("users", id));
}

get("/users",  "list_users");
post("/users", "create_user");
body_schema({"name": "str", "age?": "int"});   // invalid body → 422, automatically

serve(8080);
```

Want the guided version? The **[Wings Tutorial](/ecosystem/wings-tutorial/)**
builds three complete apps step by step — REST, token auth, and a SQLite-backed API.
